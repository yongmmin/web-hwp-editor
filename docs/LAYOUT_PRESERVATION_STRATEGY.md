# Layout Preservation Strategy

## 목표

한글 원본(`.hwp`, `.hwpx`)과 웹 편집 결과 사이의 레이아웃 차이를 줄인다.
핵심 기준은 다음과 같다.

- 페이지 흐름(페이지 분리 위치)
- 표 구조/열폭/행 병합(`colspan`, `rowspan`)
- 이미지 크기/정렬/문단 내 배치
- 문단 정렬/기본 서식

## 확정된 방향 (2026-03-27)

다음 방식으로 진행한다.

- 현재 React + TipTap 편집기 구조를 유지하되, 렌더 레이어를 3모드로 분리:
  - `원본 보기(읽기 전용)`
  - `편집 보기`
  - `비교 보기(원본/편집 분할)`
- `hwpLegacyParser.ts`, `hwpxParser.ts`를 고도화하여 변환 편집 품질을 개선
- 원본 보기 레이어는 “편집 안정성”보다 “원본 시각 보존”을 우선
- 편집 보기 레이어는 “수정 가능성/내보내기 안정성”을 우선

## 단계별 실행 계획

### 1) 원본 보기 레이어 고도화

- 페이지/표/이미지 배치 보존을 위한 읽기 전용 렌더 품질 우선 튜닝
- 원본 보기에서 편집기 기본 CSS 간섭 제거(테이블 기본 border/padding, line-height 등)
- 원본 대비 차이가 큰 샘플 문서 기준으로 시각 회귀 점검

### 2) 편집 레이어 정확도 개선

- GSO 이미지 레이아웃(크기/배치) 추출 보강
- 표 셀 위치/병합/열폭 정규화
- ~~페이지 분리 컨트롤(`pghd`, `nwno`) 처리~~ ✅ (섹션 간 자동 분리 포함)
- ~~제어 문단(`secd`, `cold`) 노이즈 제거~~ ✅ (텍스트는 보존, 제어 노이즈만 제거)

### 3) HWPX 레이아웃 일관성 강화

- section/page 관련 XML 속성 반영
- 표/이미지 배치 속성 추가 매핑
- 인라인 서식 누락 케이스 보강

### 4) 회귀 방지

- 대표 실문서 스냅샷 기반 비교 테스트 추가
- 파싱 지표(이미지 수, 표 수, 비정상 span 수 등) 자동 점검

## 비목표

- 한글 워드프로세서 엔진과 완전 동일한 WYSIWYG 보장
- 한글의 모든 객체 타입(OLE/차트/수식/복합 도형) 100% 동일 렌더

## 현재 상태 (2026-03-30 기준)

### 완료된 파서/렌더러 수정

- **텍스트 손실 수정**: 테이블 포함 문단, `secd`/`cold` 문단의 텍스트 버림 버그 수정
- **TAB 렌더링 수정**: `0x09` 확장 제어에서 `\t` 출력 + 14바이트 파라미터 스킵
- **chars 24-31 logicalPos 처리**: charShape 위치 매핑 정확도 개선
- **페이지 분리 개선**: `nwno` 추가, 섹션 경계 자동 페이지 분리
- **실제 페이지 레이아웃 반영**: `PAGE_DEF`를 읽어 페이지 크기와 본문 여백을 `pageLayout`로 보존하고, 편집 페이지에 실제 HWP 여백을 적용
- **글꼴 반영 강화**: `TAG_FACE_NAME`을 읽어 span 단위 `font-family`를 출력해 줄바꿈 위치와 셀 높이 오차를 줄임
- **문단 여백/들여쓰기 보정**: HWP의 doubled margin/indent 값을 바로잡고, 음수 indent는 `padding-left`로 분리해 셀 바깥으로 튀는 문단을 완화
- **표 앵커/배치 보정**: 인라인 표는 `inline-block`으로 유지하고, 블록 표는 `hRelTo/hAlign/x/margin`을 이용해 `pyhwp`와 같은 방식으로 `margin-left` 계산
- **표 열폭/행높이 계산 고도화**: `LIST_HEADER`의 `width/height`, `colspan/rowspan`, 셀 padding/border를 이용해 열/행 track을 푼 뒤 `pt` 단위 `colgroup`/셀 폭으로 렌더
- **TipTap 표 폭 보존**: `data-hwp-col-widths`를 테이블 노드 속성으로 보존하고, 에디터 렌더 시 `colgroup`를 다시 생성해 `setContent()` 이후에도 열 폭이 유지되게 함
- **비정상 셀 데이터 방어 로직 추가**: 깨진 `colSpan/rowSpan/width/height`를 정규화해 `200열` 확장이나 대량 빈 filler 셀 생성 방지
- **CSS 기본값 정비**: `line-height:1`, 표/셀 `content-box`, 표 `max-width:100%`로 파서 인라인 스타일과의 간섭 축소

### 품질 회귀 기준 (11개 샘플, 2026-03-30)

- 평균 점수: **0.5834** (구조 유사도 — 텍스트 길이, 표/행/셀/이미지/문단 수, 페이지 분리)
- 측정 방법: `npm run regression:hwp -- <dir>` (hwp5html 기준 비교)
- 주의: 이 점수는 HTML 구조 유사도이며 시각적 품질과 1:1 대응하지 않음

### 실문서 검증 메모 (2026-03-30)

- 대상: `2025 아트코리아랩 기술 융합 오픈이노베이션_지원신청서(데이터킹).hwp`
- 확인 결과:
  - 페이지 콘텐츠 폭 초과 테이블 `0건`
  - 비정상 `100+`열 테이블 `2건 → 0건`
  - `&nbsp;`만 있던 filler 셀 `4484개 → 0개`
  - 전체 `td` 수 `5293 → 809`
- 해석:
  - 구조 회귀 점수는 큰 변화가 없어도, 실제 시각 품질은 표 폭/빈 셀 측면에서 유의미하게 개선됨

### 남은 주요 레이아웃 과제

- 이미지 크기/배치 정확도 (GSO HWPUNIT → px 매핑 정밀화)
- 셀 내부 실제 줄 수 기반 행 높이 보정 (`PARA_LINE_SEG` 활용 검토)
- HWPX 경로에도 동일한 table track/anchor 보정 적용
- HWP 바이너리 머리글/바닥글 파싱
